关于Lambda的理解：

使用环境，有一个函数式接口，该接口的对象被作为形参传入了某个方法，在这个方法中调

用了这个接口的抽象方法。那么如何如何使用该方法，首先将Lambda看成一坨，作为一个对

象，传入某个方法的形参，在这个方法中使用这个传入的形参调用了接口唯一抽象方法,而

Lambda就是作为这个接口new的对象来调用方法。此时，再将Lambda一分为二的看待，后面

的大括号的语句作为函数式接口的唯一抽象方法的方法体来执行。

Lamba中的方法具有返回值时，其返回的值是传给将抽象接口对象作为形参的那个方法，而

关于如何使用这个返回值，取决于方法体中的执行语句。

关于方法引用

方法引用的出现正是因为Lambda中只有一条执行语句。使用方法引用进一步的简化代码。

以下几种分类：

1.引用类方法：即使用static修饰的方法

类名::方法名 就完成了类方法的引用

2.引用实例方法：创一个对象，作为第一参数，后面的是方法的形参

当然其限制条件是：首先是可以用Lambda表达式进行替换，而且该Lambda只有一条语句。

其次，使用方法引用必须要有一个形参，返回值和功能完全一致的方法供你引用，比如你需

要将一个数字字符串转换为整型数字，那么恭喜你，恰好有这样一个方法valueOf提供了这个

功能，此时不要直接使用这个方法，而是写一个函数式接口，然后写一个方法，将这个接口

的对象作为形参传入这个自定义的方法，在这个方法中，调用调用接口的抽象方法，最后在

主方法中new一个对象，用对象调用自定义方法，方法的形参传入使用极为先进的方法引用，

方法的引用根据方法的类型不同有不同的引用方式，valueOf由于是类方法，所以使用类名来

调用，Integer::valueOf，这样就完成了方法的引用，是不是显得极为简洁（浪催的）。
