Java—-集合

集合概述

集合类主要用来存储和保存数据，但与数组不同的是，数组中既可以保存基本数据类型也可

以保存对象，而集合只能保存对象。（保存的都是引用型变量）

集合分为Collection和Map两大类型。二者是集合框架的根接口。

Collection分为Set，Quene，List。

其中Set是无序的，不能保存相同的元素，所以只能通过元素本身进访问。List是有序的，

它通过索引来查找具体的位置。 Quene是一个队列。

Map集合，它保存的元素是由key和value组成的，可以通过key值来查找具体的元素位置。

Set集合

继承了Collection，与collection不同的是，set集合用add加入同样的元素时，返回值

时false。且新元素不会被加入。

HashSet类是Set集合的一个实现类特点如下：

不能保证元素排列顺序，可能与添加的顺序不同，顺序也可能发生变化。

HashSet不是同步的，多线程同时访问时，要用代码保持同步。

集合元素可以时null。

底层是一个hash表，查询的速度快。

hash值：

hashCode方法有两大类，第一种时Object中的使用对象的地址值来计算出的一种逻辑地址。

由于每一次new出的对象都有不同的地址，这就导致有时候相同属性值的对象会被重复的存

入Set集合中，于是就有了第二种重写hashCode和equals方法的操作，用来排除相同属性值

对象。HashSet中存入一个元素时，系统会调用hashCode()方法得到对象的hashCode值，由

hashCode值来确定元素的位置，如果两个元素是的equals()方法比较结果为true，但hashCo

de的返回值不同，HashSet会将其放在不同的位置存储。即判断hash中元素是否相同 1是eq

uals()的返回值为true，2是hashCode的返回值也相等。

如果重写了equals()方法那么hashCode()也应重写，能够让equals返回true的两个元素，

hahsCode的返回值也是 一样的。如果不一样，会使得set集合中保存了两个相同的元素，与

set规则相冲突。当hashCode相同但equals返回不为true，就会把这个元素放在相同的hashC

ode元素后形成链表，当链表的长度大于8时就会用红黑树代替。一般情况应该重写两个方法

以避免。hash的原理就是通过hashcode来确定元素的位置，数组虽然有索引，但其索引连续

且长度固定。

hashCode方法对hashset速度很重要。一般由以下的重写规则：

  1. 同一个对象多次调用hashCode方法返回值相同；
  2. 两个对象通过equals方法返回true时，hashCode返回值应当相同

对象中用作equals方法比较标准的实例变量应用于计算hashCode值

LinkedHashSet类

是hashSet的子类，同时使用链表维护元素的次序。在遍历时会以添加是的顺序访问。底层是

由一个链表组成。特点具有查询慢，增删改块。但即便它是有序的，但它不能元素重复。

TreeSet类

TreeSet是SortedSet的实现类，可以让元素保持排序状态

TreeSet的额外方法：

Comparator comparator(): 如果使用了定制排序，方法返回定制排序使用的comparator；如

果使用了自然排序，就返回null。

TreeSet的顺序是按元素大小排序，不是元素的插入排序

自然排序

自然排序就是调用compareTo(Object o)方法比较两个元素的大小。然后将集合按升序列。

Java提供了Comparable接口，其中的compareTo(Object obj)方法返回一个整数值，实现

该接口必须重写该方法。返回一个正整数就表示前面的大，返回一个负整数表示后面的大。

一些实现了该接口的常用类：

BigDecimals BigInteger及所有的数值型对应包装类。

Character：按字符的unicode比较大小

Boolean：true对应的包装类实例大于false对应的包装类实例

String：按字符串中字符的Unicode值比较

Date，Time：后面的时间，日期比前面的时间，日期大

把一个对象添加到TreeSet，该类的对象必须实现Compare接口，否则会出现异常。

由于在进行自然排序时必须要时同一个类的实例进行比较大小，因此传入TreeSet的元素必

须是同一个类的实例。否则会引发ClassCastException。

将对象加入TreeSet集合时，先调用对象中的compareTo方法，根据红黑树确定存储位置，

如果compareTo方法的比较值相等，该元素将不会加入集合。

compareTo的返回值TreeSet集合比较两个元素是否相等的唯一标准。

集合里存放的时引用型变量，不是真正的元素，所以通过重写的compareTo和equals方法

将两个相同的元素放入集合后，一旦一个的属性值发生改变，另一个也会发生改变。

定制排序

自然排序是根据元素的大小，集合将他们升序排列， 若要实现定制排序例如降序排列，通

过在自定义的类中实现Comparator接口，该接口含有一个int compare(T o1,T o2)方法，用

于比较哦o1和o2的大小，当返回值是正整数，o1大于o2。

如果使用定制排序就需要在创建TreeSet时，创建一个关联的comparator的对象，由这个对

象来决定排序，由于comparator是一个函数式接口，因此可以使用Lambda表达式。

当然使用Comparator比较两个时，依然不能向集合中添加对象。

泛型

泛型入门：

Java集合的缺点是将一个元素放入集合后，该集合就会忘记元素的数据类型，统一的变成

Object的编译类型。所以使用泛型来让java记住该元素对象的数据类型，防止在进行强制类

型转换时发生类型转换异常。在集合类名后面的<>中填入数据类型，表示该集合只能存入该

类对象。

深入泛型

泛型指的是允许在定义类，接口，方法时使用类型形参，这个形参在声明对象，创建对象，

调用方法时动态的指定。任何类和方法都可以使用泛型声明。

使用泛型在声明类的时候，可以生成无数个逻辑子类，当传入的泛型类型改变了就会让该类

的生成一个子类，但该类并不存在于任何位置，只是逻辑上的一种子类。

类型通配符

在声明一个泛型类后，在其创建对象或者声明对象时就必须要为其泛型传入一个泛型形参，

否则就会报泛型警告。当泛型类中传入的不同泛型间具有继承关系，这两个泛型类之间没

有任何的继承关系。

使用类型通配符，类型通配符就是一个？，例如List就是使用了泛型，让这个List成为了所

有泛型类List集合的父类，但是并不能把元素添加到该集合。类型通配符的上限和下限：

List<? extends 类型>：表示该通配符必须继承自后面的类型

List<? super 类型>：表示该通配符的类型必须是后面的类型的父类

集合中的可变参数

在面向对象中介绍了可变参数，在集合中由如下几种应用：

Arrays工具类中有一个方法：

public static List asList(T… a):返回一个由指定数组支持的固定大小的列表

List<String> list = Arrays.asList("hello", "world", "java");

返回的集合不能作增删操作，但可以修改

List接口中有一个静态方法：

public static List of(E… elements):返回包含任意数量元素的不可变列表

List<String> list = List.of("hello", "world", "java", "world");

返回的列表不能增删改

Set接口中有一个静态方法：

public static Set of (E… element):返回一个包含任意数量元素的不可变集合

Set<String> set = Set.of("hello", "world", "java");

给元素时，不能给重复的元素

返回的集合不能作增删，且没有修改的方法。
