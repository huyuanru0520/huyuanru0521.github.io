Java面向对象—-方法详解

方法的所属

方法可以看作为传统意义的函数演变而来，但有着本质上区别的是，方法必须依托一个类或者一个对象来定义，没有单独定义一个方法功能。但函数可以单独定义，整个软件可以由函数构成。但Java中类是第一的，其包含了方法，变量，还可以创建一个实例。static修饰的方法是属于类的，而未被修饰的方法只属于实例。（当涉及变量时可以看出不同。）

类中的方法互调有以下情形：

一个方法调用一个普通方法时，默认使用this作为调用者。

一个方法调用一个静态方法时，默认使用类作为调用者。这也就导致不同的对象调用static修饰的方法时总是获得

相同的执行结果。就是因为默认使用了类作为调用者。相反，没有static修饰就会得到各不相同的结果。

方法的传参

方法的传值是“值传递”。将实参的值传给形参，参数本身并不会改变。而这并不区分引用类型和基本类型

例如一个方法是将两个传进来的参数值相互交换，并且他的返回值是void型，此时打印俩个实参的变量就会看到他

俩的值并没有发生互换。这就是值传递的一个表现。

内存中的情况：

系统会自动为main方法分出一块栈内存，当main调用了一个互换的方法时就会再申请一块新的栈内存。而两个栈

内存都会出现各自的变量。因此当栈2中的变量值发生交换时，栈1并没有改变。
