Java 面向对象—-关于对象

对象的产生和使用

创建对象：

Student stu = new Student();

此处的stu即为对象。未被static修饰的方法或变量只能通过对象来调用，而static修饰的既可以被对象调用，也可

以被类调用。

当调用的方法有形参是，必须为形参赋值。类和对象，类是一般的抽像的，而对象是具体的。使用类创建对象，可

以重复的使用一般的特征。

使用new来创建对象，所需的内存都是在堆内存中。而左边的变量是一个引用型变量，其在栈内存中指向堆内存中

的对象。Student stu1 = stu；即引用型引用了一个引用型，他们都指向了同一个堆内存地址。而一个对象一旦没

有一个变量指向它，Java就会以垃圾处理回收内存。那么把一个引用变量的值赋为null，就会让这个对象变成垃

圾。

关于this关键字：this总是指向调用该方法的对象。

两种不同的引用情形：

构造方法中正在创建的对象

在方法中引用调用该方法的对象

public class Dog{
  public void Jump（）{
    System.out.println("执行jump方法");
  }
  public void Run(){
    this.Jump();//用this，当new一个对象时，this将会指向调用Run方法的对象
    System.out.println("正在执行Run方法");
  }
}

ps： Java允许对象的一个成员调用另一个成员，因此上面方法前的this也可以省略

static 修饰的方法，可以被类或实例引用，一旦用this关键字，这个关键字可能无法指向合适的对象。所

以static 修饰的对象不能用this引用，

没有static修饰的方法只能由实例引用，一旦出现在static修饰的方法中，而static修饰的方法可以由

类调用，这样没有static修饰的普通方法就可能由类来调用，此时与第一句矛盾。所以static修饰的方法

不能调用普通方法.即构成了一句======“静态成员不能直接访问非静态成员”

static修饰的成员属于类本身，完全不属于该类的实例。static修饰的成员就不应该由对象来调用，应该用类来调

用。

在静态方法中调用另一个普通方法需要创建一个对象。一般情况，普通方法访问其他方法成员变量时无需使用

this，但如果方法中的局部变量和成员变量重名，方法中又需要访问这个成员变量就必须使用this。

构造器中的this指向正在初始化的对像。而构造器中访问其他成员变量和方法时可以省略this关键字。但当构造发

方法中的变量和成员变量相同时，就可能需要this。

将this作为返回值，可以连续调用相同的方法，返回了一个对象，对象可以调用方法，当调用的方法还是该方法时就可以再返回一个对象。可以一直下去
